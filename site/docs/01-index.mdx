Include static assets in your compilation even if they aren't referenced in scripts or stylesheets

### [bud.define](/docs/bud.define)

Replace variables in your application code and templates at compile time

### [bud.dev](/docs/bud.dev)

Configure the development server

### [bud.devtool](/docs/bud.devtool)

Configure sourcemaps

### [bud.entry](/docs/bud.entry)

Specify a set of assets to include in the compilation

### [bud.env](/docs/bud.env)

bud includes support for utilizing dotenv values (in your config, templates and the client)

### [bud.externals](/docs/bud.externals)

Specify a non-standard resolution strategy for modules with a matching name

### [bud.get](/docs/bud.get)

Multi-compiler management utility allowing reference of child compilers

### [bud.hash](/docs/bud.hash)

Enable or disable filename hashing of built assets

### [bud.hooks](/docs/bud.hooks/index)

Modify internal values with callbacks

### [bud.make](/docs/bud.make)

Create a new, configurable instance of Bud

### [bud.minimize](/docs/bud.minimize)

Enable or disable minification of static assets

### [bud.mode](/docs/bud.mode/bud.mode)

CSpecify if a build is being run in a production or development context

### [bud.path](/docs/bud.path)

Get an absolute path for a given Framework location

Make a variable/module available throughout the entire application |
